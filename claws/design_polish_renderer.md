You are rendering a **polished, photographic-quality mockup** of a
Home Center design concept.

You are given a **reference image** — a structural grayscale-ish
wireframe of the same concept generated by our structural renderer. Your
job is to transform it into a production-quality rendering of the Home
Center TV dashboard, **preserving the reference's exact regions,
hierarchy, proportions, and labels**.

This is a polish pass. Not a reinterpretation.

## Hard constraints

- **Do not invent new regions.** Every card / panel / overlay in the
  reference must appear in your output. No extras.
- **Do not change proportions.** If the reference shows the primary
  region taking 2/3 of the screen width, yours must too.
- **Preserve the reading order.** Top-left, top-right, bottom strip —
  whatever the reference dictates.
- **Stay at 16:9-ish.** Output should feel like a 1920×1080 TV
  dashboard. Edges/crop should not leak into the content.
- **Family context, not corporate.** This is a home kitchen TV, not a
  CRM dashboard.

## Home Center design language

- Near-black background (`#0A0A0A`).
- Geist / Inter sans-serif for UI; JetBrains Mono for numeric data
  (times, temperatures, counters).
- Primary text white; secondary `#FFFFFFAA`; tertiary `#FFFFFF66`.
- Accent blue `#60A5FA` for primary actions / brand.
- State tints used sparingly: amber `#F59E0B` for warnings/overlays,
  red `#EF4444` for urgent, green `#4ADE80` for affirmative, blue
  `#60A5FA` for info.
- Cards: 1px subtle white border (`#FFFFFF30`), 8-12px rounded
  corners, transparent interior or very faint fill. Overlay cards use
  the amber chrome (faint amber bg + amber border).
- Interruptive banners have a dashed amber border.
- No marketing flourishes. No logos. No vendor photos.

## Content guidance

Populate the regions with plausible, home-plausible dummy data —
matching whatever the reference's labels imply:

- Calendar rows like `07:45  Drop Lucy at school`, `10:30  Dr. Shaw`.
- Weather with a large temperature, condition, feels-like, wind,
  humidity, H/L. Realistic numbers (e.g. 54°F / Partly Cloudy).
- Birthdays as name + `Apr 26 · 5 days` + a small pill button.
- Morning checklist items with checkboxes, names (Lucy, Ivy, Peter,
  Andrew), and time eyebrows.
- Claw Suggestions: icon tile + title + secondary subtitle + chevron.
- Photos (if present): 2×2 or 3×2 grid of realistic family photography
  — candid, warm, muted color grading. No stock-photo look.
- Bedtime toast: small amber-chromed overlay if the reference shows
  one.

## Type + layout fidelity

- Labels match the reference: if the reference says `MORNING
  CHECKLIST` as an eyebrow, yours says the same.
- Numeric data (temps, times) uses monospace.
- Hierarchy is the primary visual weight signal — don't use size to
  "fix" a region that the reference intentionally kept compact.

## Absolutely no

- No text overlay saying "mockup" or "concept" or "draft".
- No watermarks.
- No caption bars explaining the image.
- No photo-realistic people's faces — prefer silhouettes, backs of
  heads, or family-warm scenes where faces are soft/small.
- No alphabet soup of mock data fields. Every element must be content
  a real Home Center screen could contain.

## Output

A single image. No text explanation, no multiple options, no
concept-pair comparison. Just the polished dashboard at 16:9.
